## 🎩 Зачем он нужен

Вступление по заявке отлично отсекает ботов и спам — но теперь кто-то должен
вручную жать «принять» под каждой заявкой. **gatekeepeer** становится на этот пост
за вас: видит каждую новую заявку и тут же её одобряет.

## ⚙️ Как это работает

1. В канале остаётся включённым вступление по заявке — для людей ничего не
   меняется, они так же нажимают «Подать заявку».
2. Бот видит каждую заявку и мгновенно её одобряет. Тихо, без сообщений в канал.
3. В любой момент можно поставить приём на паузу — заявки начнут копиться, пока
   вы не включите бота снова.

> [!NOTE]
> Бот принимает только **новые** заявки, поданные после его подключения. Заявки,
> висевшие до этого, Telegram не отдаёт ботам — их придётся обработать вручную.

## 🚪 Подключение канала

1. Найдите бота в Telegram `@gatekeepeerbot` и нажмите **Start**.
2. В настройках канала включите **вступление по заявке**
3. Добавьте бота в **администраторы** с правом **«Добавлять участников»**.

Готово — бот сам обнаружит канал и заступит на пост.

## ❓ Частые вопросы

**Бот пишет что-то в канал?** Нет. Он только одобряет заявки, молча.

**Что если поставить приём на паузу?** Новые заявки просто остаются висеть — вы
примете их вручную или снова включите бота.

**Кто-то ещё из админов сможет менять настройки?** Нет, только создатель канала.

**Можно подключить несколько каналов?** Да, бот мультиарендный — каждый видит и
настраивает только свои.

<details>
<summary>🛠 Для разработчиков (самостоятельный хостинг)</summary>

<br>

Стек: **Bun · TypeScript · [grammY](https://grammy.dev) · libSQL
([Turso](https://turso.tech))**, апдейты через webhook.

```sh
bun install
cp .env.example .env   # BOT_TOKEN, WEBHOOK_URL, WEBHOOK_SECRET
bun run dev            # вебхуку нужен публичный HTTPS — поднимите туннель
```

В разработке БД — локальный SQLite-файл; для прода задайте `TURSO_DATABASE_URL`
и `TURSO_AUTH_TOKEN`. Деплой — Fly.io + Turso, подробности в
[`fly.toml`](./fly.toml) и [`Dockerfile`](./Dockerfile).

Команды: `bun start`, `bun test`, `bunx tsc --noEmit`.

</details>
