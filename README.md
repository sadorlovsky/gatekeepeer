# gatekeepeer

Бот, который сам принимает заявки на вступление в ваш приватный Telegram-канал
или группу. Включили приём по заявкам, чтобы отсеять ботов и спам — но теперь
кто-то должен вручную жать «принять» под каждой заявкой? Добавьте gatekeepeer, и
он будет делать это за вас, мгновенно и круглосуточно.

## Как это работает

1. В канале остаётся включённым вступление по заявке — для людей ничего не
   меняется, они так же нажимают «Подать заявку».
2. Бот видит каждую заявку и тут же её одобряет. Тихо, без сообщений в канал.
3. Вы в любой момент можете поставить приём на паузу — заявки начнут копиться,
   пока вы не включите бота снова.

## Подключение канала

1. Найдите бота в Telegram и нажмите **Start**.
2. В настройках канала включите **вступление по заявке**
   (Пригласительные ссылки → «Заявки на вступление»).
3. Добавьте бота в **администраторы** канала с правом **«Добавлять участников»**.

Всё — бот сам обнаружит канал и начнёт принимать заявки.

## Управление

Всё управление — в личном чате с ботом. Вы видите только свои каналы.

| Команда     | Что делает                                          |
| ----------- | --------------------------------------------------- |
| `/channels` | список ваших каналов и кнопки вкл/выкл авто-приёма   |
| `/status`   | подключён ли бот и включён ли приём                  |
| `/stats`    | сколько заявок уже принято                           |
| `/help`     | краткая справка                                      |

Владельцем настроек канала становится его **создатель** — со-админы не могут
перехватить управление.

## Частые вопросы

**Бот пишет что-то в канал?** Нет. Он только одобряет заявки, молча.

**Что если поставить приём на паузу?** Новые заявки просто остаются висеть —
вы примете их вручную или снова включите бота.

**Кто-то ещё из админов сможет менять настройки?** Нет, только создатель канала.

**Можно подключить несколько каналов?** Да, бот мультиарендный — каждый видит и
настраивает только свои.

<details>
<summary>Для разработчиков (самостоятельный хостинг)</summary>

Стек: Bun · TypeScript · [grammY](https://grammy.dev) · libSQL
([Turso](https://turso.tech)), апдейты через webhook.

```sh
bun install
cp .env.example .env   # BOT_TOKEN, WEBHOOK_URL, WEBHOOK_SECRET
bun run dev            # вебхуку нужен публичный HTTPS — поднимите туннель
```

Деплой — Fly.io + Turso, подробности в [`fly.toml`](./fly.toml) и
[`Dockerfile`](./Dockerfile). Команды: `bun start`, `bun test`,
`bunx tsc --noEmit`.
